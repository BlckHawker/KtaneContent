モジュール詳細：死刑執行スイッチ

お…おおおいおい、何をあわわ慌てているんだ？どうせミスするんだから、ししししし心配しないで。

3つのボタンのどれかを押せば、モジュールは解除される。ボタンのひとつは「死刑執行スイッチ」であり、これを押すと、以降のどこかのタイミングでミスが記録される。*

それぞれのボタンについて読み順に、以下の表を使用し𝑥nの値を割り当てる。

𝑥1 𝑥2 𝑥3
バッテリー、 モジュロ 3 インジケーター、 モジュロ 3 ポート、 モジュロ 3
(モジュロ 3: 3で割った余りを求める。 例: 11 ÷ 3 = 3 あまり 2、つまり 11 モジュロ 3 = 2。)

また、それぞれのボタンの上にある十六進数ラベルの合計を取得し、それを三進数に変換する。これを、Dと呼ぶ。(詳細は付録 B451C を参照すること)

最初に適用されるルールを参照し、死刑執行スイッチを見つける:

(ボタン1が一番左、ボタン3が一番右)

  1. 𝑥nの3つの値がすべて等しい場合、死刑執行スイッチはボタン(𝑥1 + 1)である。
  2. そうでなければ、𝑥nの3つの値がすべて異なる場合:
    1. これらの値のうち、ちょうど1つの値がD内の数字に現れる場合、死刑執行スイッチはこの値が割り当てられたボタンである。
    2. そうでなければ、これらの値のうち、ちょうど1つの値がD内の数字に現れない場合、死刑執行スイッチはボタン(その値 + 1)である。
    3. それ以外の場合、死刑執行スイッチはボタン(Dの下一桁 + 1)である。
  3. そうでなければ、𝑥nの最大値が他と重複していない場合、死刑執行スイッチは
    ボタン((Dの下一桁 + その値) モジュロ 3 + 1)である。
  4. そうでなければ、𝑥nの最小値が0である場合、死刑執行スイッチは
    ボタン((Dの上一桁 + Dの下一桁) モジュロ 3 + 1)である。
  5. それ以外の場合、死刑執行スイッチはボタン(Dの上一桁 + 1)である。

*これ以前に爆弾上のモジュールがすべて解除されている場合、モジュールには即座にミスが記録される。

付録 B451C: 十六進数と三進数

我々が日常的に使っている数字体系は十進数(または十進法)と呼ばれる。0から9までの10個の記号をそれぞれ使用し、位取りで数字を表す。一番右の位は一を意味し、その左の位は十、といった具合に、左の各位は前の位の10倍の意味を持つ。それぞれの位は10の何乗かを表している。

二進数もよく見られる数字体系であり、十進数とほぼ同じ働きをするが、記号は10個ではなく0と1の2個である。二進数の位の値は2の累乗であるため、左のそれぞれの位は十進数で見られる10倍ではなく、2倍になる。

十六進数と三進数は同じ公式に従う — 十六進数には16個の数字があり、三進数には3個の数字がある。 十六進数は16の累乗を使用し、三進数は3の累乗を使用する。

十六進数で使用する16個の数字は、最初の10個を0〜9、最後の6個をA〜Fとする。例えば、十六進数の9Eは(9 × 16) + (14 × 1)に等しく、十進数では158となる。三進数の210は(2 × 9) + (1 × 3) + (0 × 1)に等しく、十進数では21となる。
